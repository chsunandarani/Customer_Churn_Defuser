# Customer Churn Defuser

**AI-Powered Customer Retention and Churn Prediction System**

Customer Churn Defuser predicts which customers are likely to churn before they
leave, scores each customer's risk, and recommends a concrete retention action
(discount, success call, tutorial, loyalty reward) so teams can intervene early.

---

## Features

- **Dashboard** — total customers, churn rate, active customers, and a
  high/medium/low risk breakdown with live Chart.js visuals.
- **Single Customer Prediction** — fill in a form for one customer and get a
  churn risk score (0–100%), risk level, and tailored recommendations.
- **Bulk CSV Upload** — upload a CSV of many customers, get predictions for
  all of them in a table, and download `predicted_results.csv`.
- **Analytics** — interactive Plotly charts: churn distribution, risk
  distribution, feature importance, churn-by-tenure trend, and customer
  segmentation (engagement vs. satisfaction).
- **ML pipeline** — Logistic Regression, Random Forest, and Gradient
  Boosting are trained and compared via cross-validated grid search; the
  best model (by ROC-AUC) is automatically selected and persisted.
- Modern, responsive, mobile-friendly **Blue + White SaaS dashboard** UI with
  sidebar navigation.

## Installation

```bash
# 1. Clone / unzip the project, then enter the folder
cd Customer_Churn_Defuser

# 2. Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt
```

---

## Dataset Generation

Generates 10,000 synthetic customer records with realistic churn logic
(low login frequency, high support tickets, low engagement, many inactive
days, and payment failures all increase churn probability):

```bash
python dataset_generator.py
```

Output: `dataset/customer_churn_dataset.csv`

---

## Model Training

Runs EDA (missing values, correlation heatmap, distributions, pairplot),
trains and grid-searches Logistic Regression, Random Forest, and Gradient
Boosting, evaluates each on Accuracy / Precision / Recall / F1 / ROC-AUC,
and saves the best model:

```bash
python train_model.py
```

Outputs:
- `churn_model.pkl`, `scaler.pkl` (used by the web app)
- EDA + feature-importance charts in `static/images/`
- `models/model_metrics.json` with full metric comparison

## Deployment (Render)

1. Push this project to a GitHub repository.
2. In [Render](https://render.com), create a **New Web Service** and connect
   the repo. Render will detect `render.yaml` automatically (or set manually):
   - **Build Command:**
     `pip install -r requirements.txt && python dataset_generator.py && python train_model.py`
   - **Start Command:** `gunicorn app:app`
3. Deploy. Render will build the dataset and model at build time, then serve
   the Flask app with Gunicorn.
4. Once live, your app will be available at the Render-provided URL
   (e.g. `https://customer-churn-defuser.onrender.com`).
